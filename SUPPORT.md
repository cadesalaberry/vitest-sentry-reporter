# Support

Thanks for using `vitest-sentry-reporter`! Here's how to get help.

## Documentation

- **[README](README.md)** — installation, configuration, what gets reported, and
  CI auto-detection.
- **[Architectural Decision Records](docs/decisions/README.md)** — the "why"
  behind key design choices.

## Questions & help

- Search [existing issues](https://github.com/cadesalaberry/vitest-sentry-reporter/issues)
  first — your question may already be answered.
- For "how do I…?" questions, start a
  [GitHub Discussion](https://github.com/cadesalaberry/vitest-sentry-reporter/discussions)
  if enabled; otherwise open an issue.

## Reporting bugs & requesting features

- 🐛 [Open a bug report](https://github.com/cadesalaberry/vitest-sentry-reporter/issues/new/choose)
- 💡 [Request a feature](https://github.com/cadesalaberry/vitest-sentry-reporter/issues/new/choose)

Please use the issue templates — they prompt for the details (Vitest version,
Sentry SDK version, configuration) that help us help you faster.

## Security

For security vulnerabilities, **do not** open a public issue — see our
[Security Policy](SECURITY.md).
